Setup and self-hosting instructions. farmOS runs on Docker, shared hosting, or managed via farmier.com. Requires PHP 8.1+ and a database.

01
Requirements check 5 min
PHP 8.1+, MySQL/PostgreSQL, Composer. Or use Docker. Minimum 256 MB RAM; 512 MB recommended for production. Run php --version to verify.
02
Clone & install via Composer 10 min
# Create new farmOS project composer create-project farmier/farmos-project myfarmos cd myfarmos # Or pull Docker image docker pull farmos/farmos:3.x
03
Database & environment config 10 min
Create a MySQL or PostgreSQL database. Copy .env.example to .env and set DB_URL and APP_KEY. For Docker, use environment variables in compose file.
04
Run installer & enable modules 15 min
drush site:install farm --account-name=admin \ --account-pass=changeme \ --db-url=mysql://user:pass@localhost/farmos drush pm:enable farm_quantity_standard farm_sensor
05
Configure API & data connections 20 min
Enable farm_api module. Authorize OAuth2 clients for SurveyStack, Field Kit, or custom integrations. Review data sharing scopes under Admin → API → Clients.

About this tool
farmOS is a web-based application for farm management, planning, and record keeping. Provides a structured, open API enabling data flows between sensors, mobile apps, and analysis pipelines. Designed for farmer data sovereignty: self-hosted or managed, with full data portability.
